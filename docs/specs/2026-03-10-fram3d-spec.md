# Fram3d Product Specification

**Date**: 2026-03-10
**Status**: Draft

---

## 1. Product Overview

Fram3d is a 3D previsualization tool for filmmakers. It provides virtual camera controls, shot sequencing, and keyframe animation in a simplified interface that speaks cinematic language — focal length, dolly, crane — not 3D software jargon.

### 1.1 Target Users

Directors, cinematographers, and storyboard artists who need to plan shots and sequences without the complexity of Blender, Unreal, or other general-purpose 3D software.

### 1.2 Core Philosophy

Cinematic language over 3D complexity. Every interaction should feel familiar to someone who works on set, not someone who works in Maya.

---

## 2. Virtual Camera System

The camera models a physical film camera with four independent capabilities: movement, lens, focus, and shake.

### 2.1 Camera Movement

| Movement | Description |
|----------|-------------|
| **Pan** | Horizontal rotation around a pivot point |
| **Tilt** | Vertical rotation around a pivot point |
| **Dolly In/Out** | Translation along camera's forward axis |
| **Truck** | Translation along camera's right axis |
| **Crane** | Vertical translation |
| **Roll** | Rotation around camera's forward axis |
| **Orbit** | Combined pan + tilt around a selected object |
| **Dolly Zoom** | Simultaneous dolly + focal length change (Vertigo effect) |
| **Reset** | Return to default position and rotation |

Default camera position is at approximate standing eye height.

All movement speeds are configurable.

### 2.2 Lens System

Physically-based lens simulation using real-world camera sensor dimensions.

- Focal length range: 14mm – 200mm
- Default focal length: 35mm
- Field of view derived from focal length and sensor height using physically accurate formula
- Smooth focal length transitions
- 9 focal length presets: 14, 18, 24, 35, 50, 85, 100, 135, 200mm
- Three zoom modes: multiplier-based zoom, direct focal length adjustment, and dolly zoom (maintains subject size while changing perspective)

### 2.3 Focus

Animated focus-on-object that frames the selected object comfortably in the viewport.

- Smooth animated transition to target
- Calculates optimal distance from object bounds and current field of view
- Frames with breathing room (not too tight)
- Triggered when an object is selected

### 2.4 Camera Shake

Procedural handheld camera effect.

- Applies subtle position and rotation offsets
- Rotation limited to X/Y (no roll)
- Configurable amplitude and frequency
- Cleanly reverts when toggled off

### 2.5 Camera Info HUD

Overlay displaying real-time camera metadata:

- Focal length (mm)
- Camera height (meters)
- Angle of view (horizontal + vertical)
- Current aspect ratio
- Camera body preset
- Lens preset

**Camera body presets** (5):

| Body | Sensor Size |
|------|-------------|
| ARRI Alexa 35 | 27.99 x 18.17mm |
| RED Komodo | 27.03 x 14.26mm |
| Sony FX6 | 23.8 x 13.3mm |
| Canon C70 | 26.2 x 13.8mm |
| Generic 35mm | 36 x 24mm |

**Lens presets** (5): Zeiss Master Prime, Cooke S4/i, Leica Summilux-C, Sigma Cine FF, Generic Prime.

## 3. Shot Sequencer

A horizontal strip of shot thumbnails for managing the shot sequence.

### 3.1 Shot Model

Each shot contains:
- Unique identifier
- Name (auto-generated: "Shot_01", "Shot_02", etc.)
- Duration (default 5 seconds, minimum 0.1 seconds)
- Focal length (per shot, default 35mm)
- Camera animation (keyframed camera path)
- Object animations (per-object keyframed paths)

### 3.2 Sequencer UI

- Horizontal scrollable thumbnail strip
- Each thumbnail displays shot name and duration
- Click to select/switch shots
- Inline duration editing
- Add Shot / Delete Shot (minimum 1 shot enforced)
- Drag-and-drop reordering with visual drop indicator

### 3.3 Object Continuity

When creating a new shot, all currently animated objects have their current transforms captured as initial keyframes in the new shot. When navigating backward, non-animated objects restore to their initial state.

## 4. Keyframe Editor

A multi-track timeline editor for keyframe animation within each shot.

### 4.1 Layout

Three regions:
- **Transport bar**: current time / total duration display
- **Timeline area**: tracks, keyframe markers, playhead, time grid
- **Status bar**: shot name and context-sensitive keyboard hints

### 4.2 Tracks

- **Camera track**: always present, yellow markers
- **Object tracks**: one per animated object, green markers, labeled with object name

### 4.3 Keyframe Data

**Camera keyframe**: position, rotation, focal length.

**Object keyframe**: position, rotation, scale.

### 4.4 Keyframe Interaction

- Click marker to select (highlighted)
- Drag marker to reposition in time (snaps to nearest 0.1s)
- Click timeline area to scrub playhead
- Delete selected keyframe (minimum 1 camera keyframe enforced per shot)

### 4.5 Animation Interpolation

- Single keyframe: static state (no interpolation)
- Multiple keyframes: smooth interpolation between values
- Position, rotation, focal length, and scale are all independently interpolated

### 4.6 Playback

- Toggle play/pause
- Real-time advancement
- Evaluates all animations (camera + all object tracks) each frame
- Stops at shot end

---

## 5. Auto-Keyframing

Automatically creates or updates keyframes when the user manually moves the camera or objects.

### 5.1 Tracked Properties

**Camera**: position, rotation, focal length.

**Selected object**: position, rotation, scale.

### 5.2 Behavior

When manual movement is detected (and not during playback):
- If near an existing keyframe (within 0.1s): **update** that keyframe
- If not near a keyframe: **create** a new keyframe at the current time

The system distinguishes manual moves from animation-driven moves and only auto-keyframes manual input.

## 6. Scene Management

### 6.1 Scene Elements

Any object with a collision shape can be a scene element. Scene elements support:
- Click-to-select
- Hover highlighting
- Selection highlighting
- Position, rotation, and scale manipulation via gizmos
- Auto-keyframing when moved

### 6.2 Selection

Single-selection model:
- Click object to select
- Click empty space to deselect

### 6.3 Transform Gizmos

Four modes:

| Mode | Gizmo |
|------|-------|
| Select | No gizmo |
| Translate | Arrow handles on X/Y/Z axes |
| Rotate | Ring handles on X/Y/Z axes |
| Scale | Cube handles on X/Y/Z axes |

Gizmo properties:
- Axis coloring: Red = X, Green = Y, Blue = Z
- Always rendered on top (never occluded)
- Constant screen-space size
- Axis-constrained drag interaction

## 7. Camera Overlays

### 7.1 Aspect Ratio Masks

Letterbox/pillarbox bars constraining the viewport to a target aspect ratio.

| Ratio | Display Name |
|-------|-------------|
| None | Full Screen |
| 16:9 | HD/4K |
| 16:10 | 16:10 |
| 1.85:1 | Academy |
| 2.35:1 | Scope |
| 2.39:1 | DCI |
| 2:1 | Univisium |
| 4:3 | Classic |

Default: 16:9. Automatically chooses letterbox vs pillarbox based on screen shape.

### 7.2 Frame Guides

All rendered within the aspect ratio bounds. All start hidden.

| Guide | Appearance |
|-------|------------|
| Rule of Thirds | 2 horizontal + 2 vertical lines dividing frame into thirds |
| Center Cross | Crosshair with center dot |
| Safe Zones | Title safe at 90%, Action safe at 93% |
