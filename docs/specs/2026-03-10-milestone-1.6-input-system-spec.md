# Milestone 1.6: Input System — Specification

**Date**: 2026-03-10
**Milestone**: 1.6
**Status**: Draft

---

- ### 1.6. Input system (Milestone)

  Mouse and keyboard control scheme that maps cinematic operations to hardware inputs. Modifier keys change what scroll does. Letter keys switch tools and toggle overlays. Every input has exactly one meaning in any given context.

  The input system is the user experience. Camera feel — the relationship between hand movement and on-screen result — lives here. Mouse controls must feel analog and proportional: a small scroll is a small dolly, a big scroll is a big dolly. Keyboard shortcuts must be instant and unambiguous: pressing a key does one thing, always, with no mode confusion.

  The core tension is context sensitivity. The viewport and the timeline are two different interaction surfaces sharing one mouse and one keyboard. An input that means "orbit the camera" in the viewport means nothing over the timeline. An input that means "scrub" on the timeline must not accidentally move the camera. The spec must define every boundary.

  *Blocked by: 1.1 (virtual camera), 1.3 (scene management — some shortcuts control objects)*

  ---

  ## Input Context Model

  The application has two input contexts. Exactly one is active at any time, determined by the mouse cursor position:

  | Context | Active when | Governs |
  |---------|-------------|---------|
  | **Viewport** | Cursor is over the 3D viewport (above the timeline panel) | Camera movement, object selection, tool gizmos |
  | **Timeline** | Cursor is over the timeline panel or shot sequencer | Scrubbing, keyframe interaction, shot selection |

  Context switches are instantaneous — moving the cursor from timeline to viewport changes the active context on the next input event. No click required.

  Some keyboard shortcuts are **global** — they work regardless of which context is active. These are explicitly marked in the reference tables below.

  ---

  - ##### 1.6.1. Mouse controls (Feature)

    ***Scroll Y with modifier keys for focal length, dolly, crane, roll, and dolly zoom. Scroll X with Cmd for truck. Ctrl-drag for pan and tilt. Alt-drag for orbit. Left click for selection.***

    **Mouse input reference — Viewport context:**

    | Input | Modifier | Action | Notes |
    |-------|----------|--------|-------|
    | Scroll Y | None | Focal length | Scroll up = longer focal length (zoom in), scroll down = shorter (zoom out) |
    | Scroll Y | Alt | Dolly in/out | Camera translates along its forward axis |
    | Scroll Y | Shift | Crane up/down | Camera translates vertically (world Y) |
    | Scroll Y | Ctrl | Roll | Camera rotates around its forward axis |
    | Scroll Y | Cmd+Alt | Dolly zoom | Simultaneous dolly and focal length change to maintain subject size |
    | Scroll X | Cmd | Truck left/right | Camera translates along its local X axis |
    | Left drag | Ctrl | Pan + Tilt | Horizontal drag = pan (yaw), vertical drag = tilt (pitch) |
    | Left drag | Alt | Orbit | Orbit around the selected object's position (or world origin if nothing selected) |
    | Left click | None | Select object | Raycast from cursor into scene; hit = select, miss = deselect |

    **Mouse input reference — Timeline context:**

    | Input | Modifier | Action | Notes |
    |-------|----------|--------|-------|
    | Left click | None | Scrub / Select | Click on timeline area = move playhead to that time. Click on keyframe = select keyframe. Click on shot thumbnail = select shot. |
    | Left drag | None | Drag keyframe | If drag begins on a selected keyframe, reposition it along the time axis |
    | Scroll Y | None | No action | Scroll over timeline does nothing — prevents accidental camera changes |

    **Functional requirements:**
    - All scroll inputs are proportional — the magnitude of the scroll delta controls the magnitude of the camera change
    - Scroll inputs are framerate-independent — the same physical scroll produces the same camera change regardless of frame rate
    - Drag inputs (Ctrl-drag, Alt-drag) are proportional to cursor displacement, not velocity
    - Modifier keys are evaluated at the moment the scroll or drag begins — changing modifiers mid-scroll starts a new input
    - Only one mouse action is active at a time — a drag in progress blocks scroll handling until the drag completes
    - Left click with no modifier in the viewport performs selection; the same click over the timeline performs scrub — determined by context

    **Expected behavior:**
    ``` python
      # scroll without modifier adjusts focal length
      .if cursor is over the viewport
      .if no modifier keys are held
      .if the user scrolls up >>
          <== focal length increases
          <== camera position does not change
          !== any camera translation occurs

      # Alt+scroll dollies
      .if cursor is over the viewport
      .if the user holds Alt
      .if the user scrolls up >>
          <== camera dollies forward along its facing direction
          <== focal length does not change

      # Shift+scroll cranes
      .if cursor is over the viewport
      .if the user holds Shift
      .if the user scrolls up >>
          <== camera moves up along world Y axis
          <== camera rotation does not change
          !== camera moves forward or sideways

      # Ctrl+scroll rolls
      .if cursor is over the viewport
      .if the user holds Ctrl
      .if the user scrolls >>
          <== camera rolls (rotates around its forward/Z axis)
          !== camera pans or tilts

      # Cmd+Alt scroll dolly zooms
      .if cursor is over the viewport
      .if the user holds Cmd and Alt simultaneously
      .if the user scrolls >>
          <== dolly zoom executes — camera moves while focal length changes inversely
          <== the apparent size of the subject at the focal point remains constant

      # Cmd+scroll X trucks
      .if cursor is over the viewport
      .if the user holds Cmd
      .if the user scrolls horizontally >>
          <== camera moves along its local X axis
          <== camera rotation does not change

      # modifier key pressed mid-drag is ignored
      .if the user begins a Ctrl-drag (pan/tilt) in the viewport
      .if the user presses Alt during the drag >>
          <== the drag continues as pan/tilt
          !== the drag switches to orbit mid-motion

      # scroll over timeline is suppressed
      .if cursor is over the timeline panel
      .if the user scrolls >>
          <== no camera movement occurs
          <== no timeline scrubbing occurs
          !== focal length changes

      # orbit uses selected object as pivot
      .if an object is selected
      .if the user Alt-drags in the viewport >>
          <== the camera orbits around the selected object's position
          <== the selected object remains at approximately the same screen position

      # orbit with nothing selected uses world origin
      .if no object is selected
      .if the user Alt-drags in the viewport >>
          <== the camera orbits around the world origin (0, 0, 0)

      # click target depends on context
      .if the user clicks on a 3D object in the viewport >>
          <== the object is selected
          !== the playhead moves
      ||> .if the user moves cursor to the timeline
      .if the user clicks on the timeline area >>
          <== the playhead moves to the clicked time
          !== any object selection changes

      # Ctrl-drag in viewport pans and tilts
      .if cursor is over the viewport
      .if the user Ctrl-drags horizontally >>
          <== camera pans (yaws) in the direction of drag
          !== camera tilts
      ||> .if the user Ctrl-drags vertically >>
          <== camera tilts (pitches) in the direction of drag
          !== camera pans
      ||> .if the user Ctrl-drags diagonally >>
          <== camera pans and tilts simultaneously
    ```

    **Modifier conflict rules:**
    ``` python
      # only the defined modifier combinations are valid
      # undefined combinations produce no action

      # Shift+Cmd on scroll — not defined
      .if cursor is over the viewport
      .if the user holds Shift and Cmd
      .if the user scrolls >>
          <== no camera movement occurs

      # Shift+Alt on scroll — not defined
      .if cursor is over the viewport
      .if the user holds Shift and Alt
      .if the user scrolls >>
          <== no camera movement occurs

      # Ctrl+Cmd on scroll — not defined
      .if cursor is over the viewport
      .if the user holds Ctrl and Cmd
      .if the user scrolls >>
          <== no camera movement occurs

      # Ctrl+Alt on scroll — not defined
      .if cursor is over the viewport
      .if the user holds Ctrl and Alt
      .if the user scrolls >>
          <== no camera movement occurs

      # triple modifier — not defined
      .if three or more modifier keys are held
      .if the user scrolls >>
          <== no camera movement occurs

      # drag modifiers do not conflict with each other
      # because they require different modifier keys (Ctrl vs Alt)
      # only one drag can be active at a time
      .if the user is Ctrl-dragging (pan/tilt)
      .if the user additionally presses Alt >>
          <== the pan/tilt drag continues
          !== the drag switches to orbit
    ```

    **Interaction with gizmos:**
    ``` python
      # gizmo drag takes priority over camera controls
      .if an object is selected
      .if the active tool mode has a visible gizmo
      .if the user clicks and drags on a gizmo handle >>
          <== the gizmo drag operates (translate/rotate/scale the object)
          !== camera pan/tilt or orbit activates
          !== the object is deselected

      # click on gizmo handle does not deselect
      .if an object is selected
      .if the user clicks on a gizmo handle >>
          <== the object remains selected
          !== raycast selection runs

      # click on empty space deselects even when gizmo is visible
      .if an object is selected
      .if the user clicks on empty viewport space (not on the gizmo, not on any object) >>
          <== the selection clears
          <== the gizmo disappears
    ```

  - ##### 1.6.2. Keyboard shortcuts (Feature)

    ***Space for playback, QWER for tool modes, F for focus, 1-9 for focal length presets, C/V for keyframes, arrow keys for scrubbing, plus toggles for overlays, guides, shake, and presets.***

    **Keyboard shortcut reference:**

    *Global shortcuts (work in any context):*

    | Key | Modifier | Action |
    |-----|----------|--------|
    | Space | | Toggle playback |
    | Left arrow | | Scrub backward one frame |
    | Right arrow | | Scrub forward one frame |
    | Home | | Jump to start of current shot |
    | End | | Jump to end of current shot |
    | G | | Toggle rule of thirds |
    | H | | Toggle camera info HUD |
    | A | | Cycle aspect ratio forward through the list |
    | A | Shift | Cycle aspect ratio backward through the list |
    | S | Shift | Toggle safe zones |
    | H | Shift | Toggle camera shake |
    | Cmd+Z | | Reserved for undo (currently no-op) |
    | Cmd+Shift+Z | | Reserved for redo (currently no-op) |

    *Viewport shortcuts (only when cursor is over viewport):*

    | Key | Modifier | Action |
    |-----|----------|--------|
    | Q | | Select mode (no gizmo, click to select) |
    | W | | Translate mode (translation gizmo on selected object) |
    | E | | Rotate mode (rotation gizmo on selected object) |
    | R | | Scale mode (scale gizmo on selected object) |
    | R | Ctrl | Reset camera to default position and orientation (undoable) |
    | D | Ctrl | Duplicate selected object |
    | F | | Focus on selected object (animated transition) |
    | C | | Add camera keyframe at current time (no-op during playback) |
    | V | | Add object keyframe at current time for selected object (no-op during playback) |
    | Delete / Backspace | | Delete the selected item — a keyframe if a keyframe is selected (clicked in timeline), an object if an object is selected (clicked in viewport). Only one type of thing is ever truly selected at a time. |
    | 1-9 | | Snap to focal length preset (1=14mm, 2=24mm, 3=35mm, 4=50mm, 5=85mm, 6=100mm, 7=135mm, 8=200mm, 9=400mm) |
    | \[ | | Cycle camera body (previous) |
    | \] | | Cycle camera body (next) |
    | \[ | Shift | Cycle lens preset (previous) |
    | \] | Shift | Cycle lens preset (next) |

    **Functional requirements:**
    - Tool mode (QWER) persists until the user selects a different tool mode — it is not momentary
    - Only one tool mode is active at a time
    - The default tool mode on application start is Select (Q)
    - Tool mode keys are suppressed when the cursor is over the timeline to prevent accidental mode switches while interacting with keyframes
    - Playback toggle (Space) works globally and is never suppressed
    - Arrow key scrubbing works globally — scrubbing is useful regardless of where the cursor is
    - Arrow keys scrub by one frame at the project's frame rate (not hardcoded to 1/24s)
    - F (focus) requires an object to be selected; if nothing is selected, F does nothing
    - C (add camera keyframe) is a no-op during playback — keyframes are only added while stopped
    - V (add object keyframe) requires an object to be selected; if nothing is selected, V does nothing. Also a no-op during playback.
    - Delete/Backspace deletes the single selected item. Only one type of thing is ever truly selected at a time: clicking a keyframe in the timeline selects the keyframe (the object is indicated but not selected); clicking an object in the viewport selects the object (the keyframe at the playhead is indicated but not selected). If nothing is selected, the key does nothing.
    - 1-9 keys directly snap to focal length presets without any modifier or chord
    - Ctrl+D duplicates the selected object; if nothing is selected, Ctrl+D does nothing
    - Ctrl+R resets the camera to defaults; this action is undoable (pushed onto the undo stack)
    - Cmd+Z and Cmd+Shift+Z are reserved for undo/redo and must not be bound to any other action (currently no-ops until the undo system is implemented)
    - Keyboard shortcuts do not repeat on hold (no key repeat) except arrow keys for scrubbing, which do repeat

    **Expected behavior:**
    ``` python
      # tool mode switching
      .if the user presses W >>
          <== translate mode activates
          <== if an object is selected, the translation gizmo appears on it
      ||> .if the user presses E >>
          <== rotate mode activates
          <== the translation gizmo is replaced by the rotation gizmo
      ||> .if the user presses Q >>
          <== select mode activates
          <== the gizmo disappears

      # tool mode persists
      .if the user presses W (translate mode)
      .if the user clicks on an object >>
          <== the object is selected
          <== the translation gizmo appears on the object
          <== translate mode is still active (not reset to select)

      # tool key ignored over timeline
      .if cursor is over the timeline
      .if the user presses W >>
          <== the tool mode does not change
          !== translate mode activates

      # space toggles playback globally
      .if playback is stopped
      .if the user presses Space >>
          <== playback starts
      ||> .if the user presses Space again >>
          <== playback stops

      # space during duration editing is suppressed
      .if the user is editing a shot duration (text field has focus)
      .if the user presses Space >>
          <== the space character is typed into the duration field
          !== playback toggles

      # arrow keys scrub by one frame
      .if the user presses Right arrow >>
          <== the playhead advances by one frame
      ||> .if the user holds Right arrow >>
          <== the playhead advances continuously at key repeat rate

      # focus requires selection
      .if an object is selected
      .if the user presses F >>
          <== the camera animates to frame the selected object
      ||> .if no object is selected
      .if the user presses F >>
          <== nothing happens

      # camera keyframe — no-op during playback
      .if playback is stopped
      .if the user presses C >>
          <== a camera keyframe is created at the current playhead time
          <== the keyframe captures current camera position, rotation, and focal length
      ||> .if playback is active
      .if the user presses C >>
          <== nothing happens
          !== a keyframe is created

      # object keyframe — no-op during playback
      .if playback is stopped
      .if an object is selected
      .if the user presses V >>
          <== an object keyframe is created at the current playhead time for the selected object
      ||> .if no object is selected
      .if the user presses V >>
          <== nothing happens
      ||> .if playback is active
      .if the user presses V >>
          <== nothing happens
          !== a keyframe is created

      # delete with a keyframe selected (clicked in timeline)
      .if the user clicks a keyframe in the timeline >>
          <== the keyframe is selected
          <== the object is indicated (highlighted) but NOT selected
      .if the user presses Delete >>
          <== the selected keyframe is removed
          !== any object is deleted

      # delete with an object selected (clicked in viewport)
      .if the user clicks an object in the viewport >>
          <== the object is selected
          <== the keyframe at the playhead is indicated (highlighted) but NOT selected
      .if the user presses Delete >>
          <== the selected object is deleted from the scene
          !== any keyframe is affected

      # delete with nothing selected
      .if nothing is selected
      .if the user presses Delete >>
          <== nothing happens

      # duplicate selected object
      .if an object is selected
      .if the user presses Ctrl+D >>
          <== the selected object is duplicated
          <== the new copy is selected
      ||> .if no object is selected
      .if the user presses Ctrl+D >>
          <== nothing happens

      # Ctrl+R resets camera (undoable)
      .if the user presses Ctrl+R >>
          <== camera returns to default position (0, 1.6, -5) and default rotation
          <== focal length resets to default
          <== the reset action is pushed onto the undo stack
          !== any scene objects are affected

      # R without Ctrl switches to scale mode
      .if the user presses R without holding Ctrl >>
          <== scale mode activates
          !== camera resets

      # focal length presets via number keys
      .if the user presses 3 >>
          <== focal length snaps to 35mm
      ||> .if the user presses 1 >>
          <== focal length snaps to 14mm
      ||> .if the user presses 5 >>
          <== focal length snaps to 85mm

      # camera body cycling
      .if the user presses ] >>
          <== the camera body advances to the next in the list
          <== the sensor dimensions update, which may change the field of view
          <== the HUD updates to show the new body name
      ||> .if the user presses [ >>
          <== the camera body moves to the previous in the list

      # lens preset cycling with Shift
      .if the user presses Shift+] >>
          <== the lens preset advances to the next in the list
      ||> .if the user presses Shift+[ >>
          <== the lens preset moves to the previous in the list

      # aspect ratio cycling
      .if the user presses A >>
          <== the aspect ratio advances to the next option in the list
          <== the mask bars update immediately
          <== the HUD updates to show the new ratio
      ||> .if the user presses Shift+A >>
          <== the aspect ratio moves to the previous option in the list
          <== the mask bars update immediately
          <== the HUD updates to show the new ratio

      # Cmd+Z and Cmd+Shift+Z are reserved
      .if the user presses Cmd+Z >>
          <== no action occurs (reserved for future undo)
          !== any shortcut activates
      ||> .if the user presses Cmd+Shift+Z >>
          <== no action occurs (reserved for future redo)
          !== any shortcut activates

      # toggle shortcuts are idempotent toggles
      .if rule of thirds is hidden
      .if the user presses G >>
          <== rule of thirds becomes visible
      ||> .if the user presses G again >>
          <== rule of thirds becomes hidden
    ```

    **Keyboard conflicts and priority:**
    ``` python
      # H vs Shift+H — modifier distinguishes the two actions
      .if the user presses H without Shift >>
          <== camera info HUD toggles
          !== camera shake toggles
      ||> .if the user presses Shift+H >>
          <== camera shake toggles
          !== camera info HUD toggles

      # A vs Shift+A — modifier distinguishes the two actions
      .if the user presses A without Shift >>
          <== aspect ratio cycles forward
          !== aspect ratio cycles backward
      ||> .if the user presses Shift+A >>
          <== aspect ratio cycles backward
          !== aspect ratio cycles forward

      # R vs Ctrl+R — modifier distinguishes the two actions
      .if the user presses R without Ctrl >>
          <== scale mode activates
          !== camera resets
      ||> .if the user presses Ctrl+R >>
          <== camera resets
          !== scale mode activates

      # D vs Ctrl+D — modifier distinguishes the two actions
      .if the user presses D without Ctrl >>
          <== nothing happens (D is unbound)
      ||> .if the user presses Ctrl+D >>
          <== selected object is duplicated
          !== any other action occurs

      # [ ] vs Shift+[ ] — modifier distinguishes the two actions
      .if the user presses ] without Shift >>
          <== camera body cycles
          !== lens preset cycles
      ||> .if the user presses Shift+] >>
          <== lens preset cycles
          !== camera body cycles

      # text input suppresses all shortcuts
      .if any text field has focus (shot name, duration, etc.)
      .if the user presses any letter or number key >>
          <== the character is typed into the text field
          !== any shortcut activates
      ||> .if the user presses Escape >>
          <== the text field loses focus
          <== keyboard shortcuts resume normal behavior

      # shortcuts during playback
      .if playback is active
      .if the user presses a tool mode key (Q/W/E/R) >>
          <== tool mode changes immediately (tools work during playback)
      ||> .if the user presses C >>
          <== nothing happens (camera keyframe is a no-op during playback)
      ||> .if the user presses V >>
          <== nothing happens (object keyframe is a no-op during playback)
      ||> .if the user presses F >>
          <== focus activates (may conflict with playback — camera animated by both)
          <== playback pauses to avoid conflict

      # shortcuts during camera focus animation
      .if the camera is animating toward a focus target
      .if the user provides any camera input (scroll, drag, or shortcut that moves the camera) >>
          <== the focus animation is interrupted immediately
          <== the user's input takes effect
    ```

  ## Interaction During Playback

  Playback creates a special interaction state where some inputs should be available and others should be blocked to prevent confusion.

  ``` python
    # camera input during playback
    .if playback is active
    .if the user scrolls or drags to move the camera >>
        <== playback pauses automatically
        <== the camera input takes effect
        <== the user can resume playback with Space

    # object manipulation during playback
    .if playback is active
    .if the user clicks on an object >>
        <== playback pauses automatically
        <== the object is selected

    # timeline scrubbing during playback
    .if playback is active
    .if the user clicks on the timeline area >>
        <== playback stops
        <== the playhead jumps to the clicked time

    # keyframing during playback is blocked
    .if playback is active
    .if the user presses C (add camera keyframe) >>
        <== nothing happens — keyframes cannot be added during playback
    ||> .if the user presses V (add object keyframe) >>
        <== nothing happens — keyframes cannot be added during playback

    # arrow keys during playback
    .if playback is active
    .if the user presses an arrow key >>
        <== playback stops
        <== the playhead moves by one frame in the arrow direction
  ```

  ## Edge Cases

  ``` python
    # modifier key released between scroll events
    .if the user holds Alt and scrolls (dolly)
    .if the user releases Alt between individual scroll ticks
    .if the user scrolls again >>
        <== the new scroll is interpreted as focal length change (no modifier)
        !== the scroll continues as dolly

    # rapid context switches
    .if the user is scrolling in the viewport (adjusting focal length)
    .if the cursor moves over the timeline boundary mid-scroll >>
        <== scroll events stop affecting the camera immediately
        !== remaining scroll momentum changes focal length

    # no selected object for object-dependent shortcuts
    .if no object is selected
    .if the user presses V, Delete, Ctrl+D, or uses the gizmo keys (W/E/R) >>
        <== V, Delete, and Ctrl+D do nothing
        <== W/E/R still change the tool mode (the gizmo will appear when an object is later selected)

    # cursor at context boundary
    .if the cursor is exactly on the pixel boundary between viewport and timeline >>
        <== the system resolves to one context deterministically (viewport wins)
        !== inputs are dropped or duplicated

    # multiple keys pressed simultaneously
    .if the user presses Q and W at the same time >>
        <== the system processes whichever key event arrives first
        <== the second key event is processed after, resulting in the second key's mode
        !== the system enters an undefined state

    # gizmo drag vs camera Ctrl-drag ambiguity
    .if an object is selected with a visible gizmo
    .if the user Ctrl-clicks in the viewport but not on a gizmo handle >>
        <== camera pan/tilt begins (Ctrl-drag)
        !== the gizmo interferes
    ||> .if the user clicks on a gizmo handle (no Ctrl needed) >>
        <== gizmo drag begins
        !== camera control activates

    # number keys in text fields
    .if a text field has focus
    .if the user presses a number key (1-9) >>
        <== the number is typed into the text field
        !== focal length preset activates
  ```
